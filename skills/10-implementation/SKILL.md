# Implementation

## Tujuan
Mengerjakan satu issue menjadi kode.

## Kapan Digunakan
Digunakan saat mulai coding per issue.

## Input
- GitHub Issue terkait
- Requirement terkait
- Design terkait

## Langkah Kerja
1. Baca issue dan requirement.
2. Buat atau gunakan branch feature.
3. Implementasikan perubahan kecil.
4. Jalankan test.
5. Update dokumentasi atau traceability jika perlu.

## Output
- Source code
- Commit
- Pull Request

## Aturan
- Kerjakan satu issue saja.
- Jangan menambah fitur di luar acceptance criteria.

## Quality Check
- Kode sesuai requirement.
- Test lulus.
- PR menjelaskan perubahan dan penggunaan AI.

## Kondisi Gagal
Berhenti jika issue tidak jelas atau acceptance criteria tidak dapat diuji.

## Human Review
Mahasiswa memeriksa hasil PR sebelum merge.
