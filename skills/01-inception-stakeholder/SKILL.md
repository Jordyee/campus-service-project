# Inception dan Stakeholder

## Tujuan
Memahami masalah, tujuan aplikasi, stakeholder, scope, asumsi, dan pertanyaan terbuka.

## Kapan Digunakan
Digunakan pada awal proyek sebelum requirements ditulis.

## Input
- `AI_Assisted_Campus_Service_Project.md`
- `CASE.md`

## Langkah Kerja
1. Baca input.
2. Identifikasi masalah utama.
3. Identifikasi aktor dan kepentingannya.
4. Tentukan scope awal.
5. Tandai asumsi dan pertanyaan terbuka.

## Output
- `docs/requirements/01-inception-stakeholder.md`

## Aturan
- Jangan membuat fakta baru.
- Tandai asumsi secara eksplisit.
- Jangan menambah fitur di luar scope tugas.

## Quality Check
- Masalah utama jelas.
- Semua aktor wajib tercakup.
- Scope dan non-scope ditulis.
- Pertanyaan terbuka dapat dijawab manusia.

## Kondisi Gagal
Berhenti jika input utama tidak tersedia.

## Human Review
Mahasiswa memeriksa apakah aktor, scope, dan asumsi sesuai instruksi tugas.
