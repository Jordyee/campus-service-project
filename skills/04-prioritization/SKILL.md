# Prioritization

## Tujuan
Menentukan prioritas requirement dan menyelesaikan konflik kebutuhan.

## Kapan Digunakan
Digunakan setelah specification dibuat.

## Input
- `docs/requirements/03-specification.md`

## Langkah Kerja
1. Baca input.
2. Kelompokkan requirement menjadi Must, Should, Could, dan Won't.
3. Tandai MVP.
4. Jelaskan konflik dan keputusan prioritas.

## Output
- `docs/requirements/04-prioritization.md`

## Aturan
- Prioritas harus mendukung MVP.
- Jangan menjadikan fitur tidak wajib sebagai syarat awal.

## Quality Check
- MVP bisa dibangun bertahap.
- Semua fitur wajib punya prioritas.
- Keputusan prioritas punya alasan.

## Kondisi Gagal
Berhenti jika specification belum tersedia.

## Human Review
Mahasiswa menyetujui prioritas sebelum desain dan issue dibuat.
