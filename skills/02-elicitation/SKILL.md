# Elicitation

## Tujuan
Menyusun pertanyaan untuk menemukan kebutuhan stakeholder.

## Kapan Digunakan
Digunakan setelah stakeholder dan scope awal dipahami.

## Input
- `docs/requirements/01-inception-stakeholder.md`

## Langkah Kerja
1. Baca input.
2. Buat pertanyaan untuk setiap aktor.
3. Kelompokkan pertanyaan berdasarkan fitur, data, proses, dan aturan.
4. Tandai pertanyaan yang wajib dijawab sebelum desain.

## Output
- `docs/requirements/02-elicitation.md`

## Aturan
- Pertanyaan harus relevan dengan scope.
- Jangan menjawab pertanyaan dengan asumsi tersembunyi.

## Quality Check
- Setiap aktor punya pertanyaan.
- Ada pertanyaan untuk alur status.
- Ada pertanyaan untuk data, keamanan, dan batasan sistem.

## Kondisi Gagal
Berhenti jika dokumen inception belum tersedia.

## Human Review
Mahasiswa menjawab atau menandai pertanyaan yang belum diketahui.
