# UI Design

## Tujuan
Membuat rancangan halaman, navigasi, form, dan wireframe.

## Kapan Digunakan
Digunakan setelah requirement dan API utama dipahami.

## Input
- `docs/requirements/03-specification.md`
- `docs/design/02-database-api.md`

## Langkah Kerja
1. Baca input.
2. Tentukan halaman utama.
3. Buat UI flow per aktor.
4. Buat wireframe sederhana.
5. Hubungkan UI dengan requirement.

## Output
- `docs/design/03-ui-flow-wireframe.md`

## Aturan
- Fokus pada kegunaan, bukan dekorasi.
- Setiap form harus punya validasi yang jelas.

## Quality Check
- Setiap aktor punya alur UI.
- Fitur wajib dapat ditemukan di UI.
- Form dan tabel sesuai data model.

## Kondisi Gagal
Berhenti jika data/API belum jelas.

## Human Review
Mahasiswa mencoba menjelaskan alur layar dari awal sampai akhir.
