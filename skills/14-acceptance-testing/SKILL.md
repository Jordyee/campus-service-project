# Acceptance Testing

## Tujuan
Menguji alur lengkap pengguna.

## Kapan Digunakan
Digunakan setelah fitur utama selesai.

## Input
- `docs/testing/01-test-plan.md`
- Aplikasi lokal atau URL deployment

## Langkah Kerja
1. Baca acceptance criteria.
2. Jalankan alur pengguna dari awal sampai akhir.
3. Catat hasil, bukti, dan masalah.
4. Tandai requirement yang lulus atau gagal.

## Output
- `docs/testing/02-acceptance-test-results.md`

## Aturan
- Gunakan data uji yang jelas.
- Jangan menandai lulus jika ada langkah utama gagal.

## Quality Check
- Alur reporter, admin, teknisi, dan manager diuji.
- Bukti hasil tersedia.
- Kegagalan ditulis dengan langkah reproduksi.

## Kondisi Gagal
Berhenti jika aplikasi tidak bisa dibuka.

## Human Review
Mahasiswa menyetujui hasil acceptance test dan mencatat keterbatasan.
