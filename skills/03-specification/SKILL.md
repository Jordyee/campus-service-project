# Specification

## Tujuan
Membuat functional requirement, non-functional requirement, user story, dan acceptance criteria.

## Kapan Digunakan
Digunakan setelah hasil elicitation cukup jelas.

## Input
- `docs/requirements/01-inception-stakeholder.md`
- `docs/requirements/02-elicitation.md`

## Langkah Kerja
1. Baca input.
2. Buat functional requirements dengan ID `FR-XX`.
3. Buat non-functional requirements dengan ID `NFR-XX`.
4. Buat business rules dengan ID `BR-XX`.
5. Buat user stories dengan ID `US-XX`.
6. Buat minimal dua acceptance criteria untuk setiap user story.

## Output
- `docs/requirements/03-specification.md`

## Aturan
- Gunakan ID requirement.
- Minimal 12 FR, 6 NFR, 5 BR, dan 10 US.
- Jangan menambah fitur tidak wajib tanpa alasan.

## Quality Check
- Semua minimum tugas terpenuhi.
- Acceptance criteria dapat diuji.
- Requirement tidak saling bertentangan.

## Kondisi Gagal
Berhenti jika kebutuhan aktor masih terlalu kabur.

## Human Review
Mahasiswa memeriksa apakah requirement realistis untuk waktu proyek.
